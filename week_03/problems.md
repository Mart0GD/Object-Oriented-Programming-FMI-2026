# Задача 1 – Статистика на текстов файл

В тази задача трябва да извличате статистическа информация от текстов файл, подобно на статистиката в текстов редактор като **Microsoft Word**.

---

## Структура за статистика

Дефинирайте следната структура:

~~~cpp
struct file_statistics {
    int total_symbols_count;            // общ брой символи
    int non_white_space_symbols_count;  // символи без whitespace
    int line_count;                     // общ брой редове
    int non_whitespace_lines_count;     // брой непразни редове
    int words_count;                    // брой думи
};
~~~

### Дефиниции:

Дума – последователност от символи с дължина ≥ 2.  
Разделител на думи – интервал (' ').  
Непразен символ – символ с ASCII код > 32 и < 127.  
Непразен ред – ред, който съдържа поне един непразен символ.  

### Имплементация:

Имплементирайте следните функции за извличане на статистика от файл.
~~~.cpp

//  Функция, която извлича броят на всички думи във файла. 
void extract_words(file_statistics& stats, std::ifstream& is);

// Функция, която извлича броят на всички символи във файла.
void extract_symbols(file_statistics& stats, std::ifstream& is);

// Функция, която извлича броят на всички непразни символи във файла.
// Непразен символ е всеки, чийто ASKII код е > 32 и < 127
void extract_non_whitespace_symbols(file_statistics& stats, std::ifstream& is);

// функция, която извлича броят на редовете във файла.
void extract_lines(file_statistics& stats, std::ifstream& is);

// Функция, която извлича, броят на непразните редове във файла.
void extract_lines(file_statistics& stats, std::ifstream& is);

// Функция, която принтира статистиката на конзолата подаден поток
void print_stats(const file_statistics& stats, std::ostream& os);
~~~

## Задача 2 - Копиране на файлове 

В тази задача от вас ще се изисква да напишете програма, която прочита n на брой аргументи от конзолата. Тези аргументи ще бъдат имената на текстови файлове. Целта на задачата е да слеете всички файлове последователно в файл с име посленият от подадените аргументи, съответстващ на последния аргумент. Под сливане се има предвид да вземете съдържанието на всеки един от файловете и да го копирате в изходния файл. В края на задачата последния последния файл ще съдържа информацията от останалите файлове. 

Примерено изпълнение:
```
Брой на файловете: 3

Файлове: Funky, Disco, Pesho

Съдържание на Funky.txt:
~~~~
I am Funky

~~~~

Съдържание на Disco.txt:
~~~~
I have been dead for 20 years
~~~~

Изходен файл Pesho.txt
~~~~
I am Funky

I have been dead for 20 years
~~~~
```

## Задача 3 - CSV Manager

**CSV** или Comma Separated Values, е формат където всички стойности са разделени със знак запетая. В тази задача се изисква да се създаде програма, която да може да обработва такива файлове.

Дефинирайте си следната структура:

~~~.cpp
struct student{
    char name[];
    char FN[];
    int age;
};
~~~

Където името е низ с по-малко от 100 букви, а факултетният номер е поредица от 10 символа във формат: 

`<цифра>MI<2 цифири за спец.><пореден номер>`

В началото на вашата програма ще се подаде име на файл, който ще е във формат: **<име на студент> <ФН> <възраст>** последвано от запетайка и нова стойност или край на файла. От вас ще се иска да реализирате следното:

~~~.cpp
// Прочита файла и връща масив от прочетените студенти.
student* read_csv(const char* filename);

// създава коректен CSV файл, със записани стойностите на студентите от масива array.
void create_cvs_from_array(const students* array);
~~~

Покажете коректността на вашата програма, като прочетете името на файл от конзолата, извлечете масив от студентите, генерирате нов файл от масива и се опитате да извлечете студентите отново, като сравните данните с оригинала!

## Задача 4 - Изпит във ФМИ

В тази задача трябва да помогнете на ваш преподавател да пресметне оценките на студентите си по всички дисциплини, които води (да разбере колко е успял да скъса и този път). Данните, които трябва да обработите са записани в два файла, съответно с имена - "Students.txt" и "Exam.txt". 

Всеки ред във файла Students.txt има следния формат - "XXXX SSSSS", където XXXX е факултетният номер на един студент, SSSSS е името на съответния студент. Факултетният номер представлява 10 буквен низ, започващ с контролен символ, последван от 2 букви "MI", код на специалността и пореден номер на студента. Името очаквайте да е с големина най-много 100 символа. 

Всеки ред във файла Exam.txt има следния формат - "XXXX GGGG SSSS", където XXXX е факултетният номер на студент, GGGG е оценката му (реално число), а SSSS е името на предмета.

Допустимо е във Exams.txt да срещнете студенти, за които нямате информация и обратното. Напишете програма, която за всеки студнет, за когото има достатъчно много информация, извежда в нов файл **Results.txt**, средно аритметичната му оценка за семестъра.

**Следете за грешен формат на входа!**
```
Примерен Students.txt
~~~
4MI0400123 Pesho Peshov
8MI0800320 Ivan Ivanov
9MI0500189 Georgi Georgiev
~~~

Примерен Exam.txt
~~~
4MI0400123 4.23 DSA
8MI0800320 6.00 IP
4MI0400123 3.01 DSA
4MI0400123 5.80 IP
4MI0400320 6.00 DSA
4MI0400111 6.00 OOP
~~~

Results.txt
~~~
4MI0400123 DSA 3.62
8MI0800320 IP  6.00
4MI0400123 IP  5.80
4MI0400320 DSA 6.00
~~~
```

## Задача 5 - Малък интерпретатор във файл

Във текстови файл на всеки ред е записана една от елдните 4 команди - DEF(X,Y), ADD(X,Y), SUB(X,Y), MULT(X,Y). Като техните действия са следните:

- DEF - присвоява стойността на Y на X, ако X не е дефинирана да се дефинира.
- ADD - събира X и Y.
- SUB - изважда X и Y.
- MULT - умножава X и Y.

Всяка от функциите, освен DEF изисква X да е вече дефинирана променлива. Имената на променливите са еднобуквени латински символи, като не се изисква да правите разлика между малки и главни букви. Всички стойности ще са само целочислени.

От вас се изисква да напишете програма, която да прочте текстови файл с име подадено на конзолата. Файлът ще започва с PROGRAM <име> на първия ред, където име е низ с по-малко от 100 символа. След това на всеки ред ще има по една команда до срещането на ред с думата END. Ако след END започва нова програма, обработете я иначе ако има множество грешки между две програми, просто ги пропуснете докато не срещнете нова прогарма или край на файла.

Създайте нов файл като при обработката на програма ако тя е коректна изпишете <име> и след това списък от всички дефинирани променливи и техните стойности, ако в програмата срещнете грешка изведете <име> ERROR.

```
Пример:

PROGRAM First 
MOV(a, 10) 
MOV(d, 5)
SUB(a, 7)
MUL(a, d)
ADD(d, a)
END
ADD(d, 4)
MOV(c, a) 
PROGRAM SecondProgram 
MOV(a, 3)
MOV(z, 5)
ADD(3, z)
END
PROGRAM ThirdProgram
MOV(a, 3)
MOV(z, 5)
ADD(z, 3)
END

Result.txt
PROGRAM First
a = 15
d = 20
SecondProgram ERROR
ThirdProgram
a = 3
z = 8
```
