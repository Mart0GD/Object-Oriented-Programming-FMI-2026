## Структури в C++, що е то?

Структурите, както масивите така и енумерациите са **съставен тип от данни**, с който можем да боравим, но с важната подробност, че тук елементите 
не е задължително да бъдат **хомогенни** (от еднакъв тип). Структурата "окомплектова" множество от различни по тип елементи, от където идва силата както, 
но и добавената сложност на този тип данни.

### Дефиниция и декларация на структури
---

Структурата има няколко разграничаващи характеристики. Запазената дума **"struct"**, която използваме да окажем типа и крайния брой разнородни по тип полета,
които можем да добавим на една структура. Самите полетата може да са както от **стандартен тип**, така и от **user defined** тип, 
обаче **вече известен на линкера тип по време на свързване**, за да може структурата да знае какъв е размера на типа и да изчисли съответно своя като последица.
Точно затова ако се опитаме да добавим като поле самата структура като тип ще ни даде грешка, защото тя все още **не знае своя размер**.  
  
Можем да дефинираме стандартна наименована структура, на която в последствие да създадем инстанции с конкретни стойности по следния шаблон:
~~~.cpp
struct my_struct {
    int val1;
    int val2;
    int arr[];

    my_struct* inst; // само по адрес, не знаем все още размера
};
~~~
  
Друг начин да дефинираме структура е чрез така наречената наречената "ненаименована" структура. Идеята й е да 
можем просто да създадем една конкретна инстанция на някакъв обект с множество типове данни, без да създавме изцяло структура. 
Можем да го използваме като глобален обект или просто като вложен в друга структура, клас и т.н
~~~.cpp
// тук пропускаме името
struct {
  int val1;
  char val2;
} foo; // това е променлива от тип структура с полета val1 и val2
~~~

### Инициализация 
--- 
  
Ако в дефиницията на структурата няма стойности по подразбиране, то при инициализация на обект от типа на структурата, всяко неинициализирано със стойност поле
получава стойността по подразбиране на типа **(ако има такава, ще видим скоро!)**.  

~~~.cpp
// Пример за инициализация на структура

struct vector2 {
    int x = 0; // един начин за задаване на стойност
    int y{0};  // друг вариант за задаване на стойност
};

int main(void){
    // стойностите са по ред на полетата в структурата
    vector2 right  {1,0};            // инициализация със стойности без знак за = (използваме къдрави скобки)
    vector2 left = vector2{-1,0};    // оказваме типа и къдрави скобки
    vector2 up   = {0,1};            // директно задаваме стойности с инициализиращ списък
    vector2 zero;                    // тук x и y ще получват стойност по подразбиране => (0,0)

    return 0;
}
~~~

### Достъп до полетата в структура 
---

Веднъж след като създадем структура и инициализираме конкретна инстанция от структурата искаме да можем по някакъв начин да достъп полетата на този обект. 
Това става посредство оператора **'.'** ако обекта е заделен на стека или е статичн/глобален и **'->'** ако обекта е динамично заделен.  
  
Синтаксисът е следния:
- `my_struct.val1` ако структурата е заделена на стека.
- `my_struct->val1` ако е динамично заделена (имаме указател към нея).

~~~.cpp
// Примерен достъп до стойности
#include <iostream>

struct vector2 {
    int x;
    int y;
};

int main(void){
    vector2  up {0,1};
    vector2* ref_up = &up;

    // Ето тук имаме declaration + definition + initialization
    struct my_other_struct {
        int val1;
        int val2;
    } *instance = new my_other_struct{1,2}; 

    std::cout << up.x << up.y;
    std::cout << ref_up->x << ref_up->y;
    std::cout << instance->x << instance->y;

    delete instance;
    return 0;
};
~~~

## Какви операции можем да извършваме със структури?

- Структурите нямат стандартно дефинирана операция за принтиране.  
- Структурите не могат да се сведат до **boolean** стойност.  
- Не можем по подразбиране да извършваме аритметични операции със структури.  

Можем обаче да присвояваме структури. При тази операция се случва копиране 1:1 или както го наричат на английски **shallow copying**. Данните се копират дословно, интегралните типове бит по бит, а указателите по адрес, което означва че при копиране `двата указателя сочат на едно и също място!!`.

~~~.cpp
// Пример за различни операции над структури
#include <stdexcept>

struct vector2 {
    int x;
    int y;
};

struct array {
    int* data;
    int  size;
};

int main(void){
    vector2 up{0,1};
    vector2 left{0,1};

    vector2 copy = up;  // инициализация с друга структура
    copy = left;        // присвояване на структура

    array arr = {new(std::nothrow) int[4], 4};
    array arr_copy = arr;  // 1:1 копие

    delete arr.data;
    arr_copy.data[0] = 1; // segmentation fault --> достъпваме зачистена памет
    return 0;
}
~~~

### Работа с функции

Можем да подадем една структура като параметър на фунцкия по слените три начина:  
  
- По стойност. Тук се извършва копиране на структурата и не получаваме оригиналният обект, всички промени, които направим по обекта са локални за областта на действие на функцията.

~~~.cpp
// Пример

struct my_struct{
    int val1 = 0;
    int val2 = 0;
};

void foo(struct my_struct param){
    param.val1 = 1;
}

int main(void){

    my_struct instance;
    foo(instance); // нямаме реална промяна по instance 
    return 0;
}
~~~

- По референция. Тук не се извършва никакво копиране, получваме направо адреса на променливата и всички промени, които направим във функцията се отразяват на входния обект.

~~~.cpp
// Пример

struct my_struct{
    int val1 = 0;
    int val2 = 0;
};

void foo(struct my_struct& param){
    param.val1 = 1;
}

int main(void){

    my_struct instance;
    foo(instance); // val1 вече е 1
    return 0;
}
~~~

- По адрес. Тук се извършва само копие на адреса на структурата, без самата нея. Получаваме достъп за промяна на самата структура, но не и адреса на който се намира тя.

~~~.cpp
// Пример

struct my_struct{
    int val1 = 0;
    int val2 = 0;
};

void foo(struct my_struct* param){
    param->val1 = 1; // не забравяйте оператор ->
    // (*param).val = 1;
}

int main(void){

    my_struct instance;
    foo(&instance); // val1 вече е 1
    return 0;
}
~~~

> Забележка - можете да пропуснете думичката struct в декларацията на параметрите - правя го от учебна гледна точка.

## Обединения (Unions)

Обединението досущ като структурата е съставен тип данни с краен брой полета от данни от различен тип (**предварително известни**), но обаче тук вместо да пазим 
памет за всяко поле на типа, обединението има размера на най-големия тип данни в полетата му, като самият union може да приеме стойност само за едно от полетата си. Това е много полезен и гъвкав инструмент, чрез който можем да боравим с хетерогенни типове данни, използвайки изключително малко памет. 

~~~.cpp
// Пример за дефиниця на union

union my_union {
    const char* str;
    int         number;
    long long   big_number;
};

int main(void){

    my_union u;
    u.str = "sss";
    // u.number = 1; // тук пишем върху общата памет на обединението, u вече не е коректно и това действие е undefined bahviour 

    std::cout << u.str << '\n';
    std::cout << u.number; // gibbrish
}
~~~  

Може би вече се досещате, че с помощта на обединенията можем да се опитаме да направим свой собствен хетерогенен контейнер, чрез който да съхраняваме данни от различни типове на едно място. Една една хитра и съвсем простичка имплементация на масив с елементи **int**, **float**, **char**
~~~.cpp
#include <iostream>

enum type { INT, FLOAT, CHAR };

struct value{
    type t;
    union {
        int i;
        float f;
        char c;
    } val;
};

struct array {
    value arr[10]{};
    int   size = 0;
};

void print_value(const struct array& arr);

int main(){
    array arr;

    arr.arr[0] = {INT, {10}};
    return 0;
};

void print_value(const struct array& arr, int index) {
    switch(arr.arr[index].t){
        case INT   : std::cout << arr.arr[index].val.i;
        case FLOAT : std::cout << arr.arr[index].val.f;
        case CHAR  : std::cout << arr.arr[index].val.c;
    }
}
~~~

## Битови полета

Битовите полета са начин да декларираме размера на поле в структура, клас или обединение с цел намаляване на размера на обектите, които ще се създадат от тези типове.   
  
Синтаксисът на Битово поле е следния:  
  
`<field> : <integral value>`

Типовете, върху които работят битовите полета са **интегралните типове** и **bool**, а ширината **не** бива да надхвърля размера на съответния тип. Много важна подробност е, че нито можем да вземем адреса на битово поле, нито можем да вземем референция към него.

~~~.cpp

// Пример 1
struct S1
{
    // Ще има размер 2 байта
    unsigned char b1 : 3; // първите 3 бита
    unsigned char    : 2; // маркираме следващите 2 бита като неизползвани
    unsigned char b2 : 6; // още 6 бита. От 6 + 5 > 8, означва че започваме 2 байт
    unsigned char b3 : 2; // още два бита
    // Остават ни 3 свободни бита, които ще се закръглят
};

// Пример 2
struct S2
{
    // 3 бита за b1
    // 5 бита се махат
    // 2 бита за b2
    // 6 бита остават неизползвани

    unsigned char b1  : 3;
    unsigned char     : 0; // анонимно поле, което започва нов байт 
    unsigned char b2  : 2;
};

// Пример 3 (Силно зависи от компилатора, моята версия на GCC, не прави нови блокове и връща 24 + 4 = 28 байта)
struct Student
{
    char name[24];

    unsigned int facNumber       : 22;
    unsigned int year            : 3;
    int stream                   : 2; // нов тип, започва нов блок, останалите 7 бита остават неизползвани
    unsigned int group           : 3;
    unsigned int interrupted     : 1;
    unsigned int stateTuition    : 1;

    // Размер: 4 + 4 + 4 = 12 + 24 = 36
};

// Пример 1 нагледно
int main(void){
    S1 s

    s.b1 = 0b111;
    s.b2 = 0b101111;
    s.b3 = 0b11;
 
    // 1110000011110111
    // └┬┘├┘└┬┘└─┬──┘└┤
    // b1 u  a   b2  b3

    // u - пропуснатите 2 бита
    // а - закръглените 3 бита от компилатора
    // Общо размер - 2 байта
    return 0;
}
~~~

## Подравняване в паметта

Поддравняването в паметта е нещо, което се извършва за по улеснено четене от процесора, тъй като той чете на буфери от 4 byte-a за 32 битови процесори и 8 byte-a за 64 битови. Затова компилаторите ни извършват едно тайно подравняване, което трябва да имаме напредвид, то зависи от няколко неща:

- Компилатора.
- Подредбата на полетата.
- Компютърната архитектура.

Най-важният факт за полетата е, че всяко трябва да започва на адрес кратен на размера му в паметта и цялата структура трябва да е кратна на най-големият по размер тип в структурата.  

~~~.cpp
// Да разгледаме няколко примера

// Пример 1
struct S
{
    char a; // всеки адрес е кратен на 1
    char b;
    // имаме padding от 2 байта тук
    //char padding[2];
    int c; // адреса вече ще е кратен на 4

    // резултат: 1 + 1 + 2 + 4 = 8, което е кратно на 4
};

// Пример 2
struct S
{
    char a; // всеки адрес е кратен на 1
    // имаме padding от 3 тук
    // padding char[3];
    int c;  
    char b; 

    // резултат: 1 + 3 + 4 + 1 = 9, което не е кратно на 4 => 12 байта 
};

// Пример 3
struct Student
{
    char name[24];

    unsigned int facNumber;
    unsigned int year;
    unsigned int stream;
    unsigned int group;
    bool interrupted;
    bool stateTuition;

    // резултат: ??
};

// Пример 4
struct Student
{
    char name[24];
    unsigned int facNumber;
    unsigned char year;
    unsigned char stream;
    unsigned char group;
    bool interrupted;
    bool stateTuition;
};
~~~
