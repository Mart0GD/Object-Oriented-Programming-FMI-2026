## Фундамент

В това упражнение ще се занимаваме с една много удобна функционалност на езика C++, а именно предифинирането на оператори. Но преди това, нека разберем защо изобщо ни е нужно нещо подобно. Нека вземем за пример следната структура:

~~~.cpp
struct Vector2D{
    Vector2D(double x, double y) : x(x), y(y) {};
    double x, y;
};

Vector2D add(const Vector2D& l, const Vector2D& r) { return {l.x + r.x, l.y + r.y}; }
Vector2D sub(const Vector2D& l, const Vector2D& r) { return {l.x - r.x, l.y - r.y}; }
double   dot(const Vector2D& l, const Vector2D& r) { return l.x * r.x + l.y * r.y;  }
~~~

Тази структура имплементира стандартни операции за манипулация и всичко е напълно валидно. Обаче възниква въпросът - какво става ако имам по сложен израз от вида: `((v1 + v2) * v3)`, като приемаме че * означава скаларно произведение? Кодът ни със сегашната си структура ще изглежда така:

~~~.cpp
int main(){
    Vector2D v1(0,0), v2(1,1), v3(13, 15);
    double res = dot(v3, add(v1,v2));
}
~~~

Това започва да става по-трудно за четене и съответно предпоставя грешки. Затова е редно да се запитаме дали не може ли да имаме просто нормален аритметичен израз, използвайки тези вектори. Искаме да получим нещо подобно:

~~~.cpp
int main(){
    Vector2D v1(0,0), v2(1,1), v3(13, 15);
    double res = (v1 + v2) * v3;
}
~~~

Отговорът на този въпрос е да, можем! Предефинирането на оператори ни позволява да придадем на някакви наши **потребителски** типове специално значение чрез стандартните оператори в езика. Като се целим нашите имплементации да са възможно най-близко до очакваното поведение на тези типове в света, който моделираме...
  
Но нека засега да дадем една стъпка назад и  разгледаме абстрактно нещата от високо...

## Какво характеризира една операция?

**Брой аргументи**

Една операция може да има произволен брой аргументи теоритично, но в езика C++ ще срещнем:

  - унарни - **!**, **++** ...
  - бинарни - **<<**, **>>**, **всички аритметични операции** ...
  - тернарни - само **?:**

Ето в този израз имаме всеки вид оператор: `a ? a + (-b) : -c`

**Приоритет**

Това са набор от математически правила, които ни гарантират ред на извършване на операциите. Като имаме приоритет сме сигурни, че в израз `1 + 2 * 4`, първо ще се изчисли ``2 * 4 = 8`` и след това ще се събере с ``8 + 1 = 9``.

**Асоциативност**

Когато в рамките на даден израз участват операции с еднакъв приоритет, за разрешаване на последователността на изпълнение на операциите се използва тяхната асоциативност. Операциите могат да бъдат **ляво** или **дясно** асоциативни.
  
Ето пример:  
В израза ``5 – 3 + 2``, операциите събиране (+) и изваждане (–) имат еднакъв приоритет. Те са ляво асоциативни, което означава, че първо ще бъде извършено изваждането ``5 – 3 = 2``, след което ``2 + 2 = 4``.

**Позиция на оператора спрямо аргументите**

Една операция в нашия контекст може да бъде една от следните:

  - префиксна - **++a**, **-a** ...
  - постфиксна - **a++**, **a->** ...
  - инфиксна   - **a + b**, **a - b** ...


## Кои оператори може/не можем да предефинираме в C++

В тази табличка изчерпателно са показани всички оператори, които могат да се предефинират в езика **C++**

| Категория                    | Оператори                                      | Бележки |
|-----------------------------|------------------------------------------------|--------|
| Аритметични (унарни)        | +, -                                           | Унарни версии |
| Аритметични (инкремент)     | ++, --                                         | Префиксни и постфиксни |
| Аритметични (бинарни)       | +, -, *, /, %                                  | Стандартни |
| Сравнение                   | ==, !=, >, <, >=, <=                          | |
| Логически                   | &&, \|\|, !                                   | |
| Побитови                    | &, \|, ^, ~, >>, <<                           | |
| Указатели                   | *, ->, ->*, &                                 | Дереференция, достъп |
| Индексиране                 | []                                             | |
| Функционално извикване      | ()                                             | functor |
| Преобразуване на тип        | (type)                                         | cast оператор |
| Управление на памет         | new, delete                                    | |
| Запетая                     | ,                                              | |

**Какво не можем да предефинираме?**  
  
Не може да се предефинират операторите следните оператори:

  - .  (достъп до член данни/функции на обект)
  - .* (достъп до член данни на обект чрез указател към тях)
  - :: (scope оператора)
  - ?: (тернарният оператор)
  - sizeof, typeid

Първите три не можем да предефинираме от логически съображения, тъй като най-верятно потенциално предефиниция ще доведе до много сериозни грешки, тъй като имплементацията им не е елементарна отдолу. 

Тернарният оператор не може да предефинираме поради факта, че само един от аргументите му се изчислява и типът на връщане не е ясен по време на компилация

Последните две, както и всички други ключови думи и препроцесорни директниви не могат да се предефинират, тъй като работят по време на компилация, единствените запазени думи, които предефинираме са new и delete, тъй като те имат runtime действие...

## Как да предефинираме операции в C++

Когато предефинираме оператори ще следваме следният шаблон:

``<тип на връщане> operator <оператор> (<аргументи>)``

**Бинарни операции**

Нека вземам като пример отново нашият клас **Vector2D**. Ще искаме да можем да съберем да вектора и като резултат да върнем нов вектор без да променяме никой от подадените.
Тогава нашият израз ``v1 + v2`` ще бъде видян от компилатора по един от три начина, в зависимост от това как сме го имплементирали:

~~~.cpp
// Можем да имаме външна функция
Vector2D operator + (const Vector2D l, const Vector2D r);

v1 + v2 --> operator + (v1, v2); 

// Можем да имаме член функция. Тъй като не променяме текущият обект, правим функцията константа 
Vector2D Vector2D::operator + (const Vector2D other) const

v1 + v2 --> Vector2D::operator(&v1, v2) или v1.operator + (v2);
~~~

Друга по-интересна възможност би била да добаим дробно число към нашият вектор, като изразът ни ще изглежда така:
  
``Vector2D + double``
  
Тук ако имаме кастинг конструктор, който създава вектор от число, компилаторът ни може имплицитно да се възползва от това и да извика събиращият оператор с хомогенни типове. Но в общият случай, когато нямаме такъв, ще трябва изрично да дефинираме логиката.

~~~.cpp
Vector2D operator + (const Vector2D v, double number){
    return {v.x + number, v.y + number};
}

// Тъй като операцията е комутативна ни трябва и обратната посока, но вече можем да се възползваме от вече готовата логика горе
Vector2D operator + (double number, const Vector2D v){
    return v + number;
}
~~~

**Кои операции дефинираме като член функции и кои като външни?**

Ето тази таблица изчерпва сравнително добре вариантите, кои операции можем да дефинираме като член функции и кои не

| Израз      | Като член-функция                | Като външна функция         |
|------------|----------------------------------|-----------------------------|
| @a         | (a).operator@()                 | operator@(a)               | 
| a@b        | (a).operator@(b)                | operator@(a, b)            |
| a = b      | (a).operator=(b)                |  cannot be non-member     |  
| a @= b (аритметичен)      | (a).operator=(b)                |  cannot be non-member     |
| a(b...)    | (a).operator()(b...)            |  cannot be non-member     |
| a[b...]    | (a).operator[](b...)            |  cannot be non-member     |
| a->        | (a).operator->()                |  cannot be non-member     | 
| @a (унарен)| (a).operator@(0)                | operator@(a, 0)            |  
|Като @ е в контекста на кой да е оператор в езика|

Като допълнително операциите за преобразуване между типове трябва да се дефинират в класа и като изключение унарният оператор **&**...

Оператори с ляв аргумент от друг тип интуитивно, не могат да бъдат член фунцкии, тях пишем само външно...

**Нека дефинираме operator += за Vector2D**

Вече си изяснихме как би изглеждала бинарната операция Vector2D + Vector2D, която не променя аргументите си. Интуитивно сега можем да напишем и операцията:

``Vector2D += Vector2D``

Видяхме в горната секция, че тази операция ще бъде задължително член функция на Vector2D. Друга съществена разлика е, че в този случай операцията ще променя обекта...

~~~.cpp
// връщаме референция към обекта, за да може да се наслагва операцията: v1 = v2 = v3...
Vector2D& Vector2D::operator += (const Vector2D& other)
{
    this->x += other.x;
    this->y += other.y;

    return *this;
}
~~~

Тъй като аритметични операции от типа += много често ръка за ръка с нормалната бинарна операция, като добра практика ще виждате често едната да се имплементира чрез другата. 
  
Ето как бихме разписали първият пример с нашия вектор, използвайки предефиниране на операции.

~~~.cpp
struct Vector2D{
    Vector2D(double x, double y) : x(x), y(y) {};
    double x, y;

    Vector2D& operator += (const Vector2D& other)
    {
        this->x += other.x;
        this->y += other.y;

        return *this;
    }
};

Vector2D operator + (const Vector2D& l, const Vector2D& r)
{
    Vector2D res(l);
    res += r;
    return res;
}

double operator * (const Vector2D& l, const Vector2D& r)
{
    return l.x * r.x + l.y * r.y;
}

int main(){

    Vector2D v1(0,0);
    Vector2D v2(1,1);
    Vector2D v3(13, 15);

    double res = v3 * (v1 + v2);
    v1 += v2;
    Vector2D v4 = v1 + v2;
}
~~~

Нека сега разгледаме различни операции, които можем да предефинираме...

### Операции && и ||

Стандартната логика на тези операции се възползва от така нареченият **short circuit evaluation**, който позволява в изрази като ``x && y``, ако първият аргумент е лъжа, вторият изобщо да не се изчислява или в изрази като ``x || y``, ако първият е истина, то вторият да не се изчислява.

Тази логика ще загубим ако напишем наши собствени имплементации на тези оператори, така че трябва да се има напредвид...

~~~.cpp
// Една проста имплементация...
bool operator && (const Vector2D& f, const Vector2D s)
{
    return f.x != 0 && f.y != 0 && s.x != 0 && s.y != 0;
}
~~~

### Операции << и >>  
  
Тези оператори винаги се пишат като външни фунцкии, тъй като левият им аргумент е поток. Запомнете, че в какъвто формат пишете в изходния поток, в такъв трябва да четете от входния...

~~~.cpp
std::ostream& operator << (std::ostream& os, const Vector2D& val)
{
    os << val.x << ' ' << val.y;
    return os;
}

std::istream& operator >> (std::istream& is, Vector2D& val) 
{
    is >> val.x >> val.y;
    return is;
}
~~~

### Унарни операции

Можем да дефинираме унарна операция **-**, която да обръща посоката на нашия вектор като външна фунцки с един аргумент или като член фунцкия, без аргументи, ще покажа само първото, защото има повече смисъл...

~~~.cpp
Vector2D& operator - ()
{
    this->x = - this->x;
    this->y = - this->y;
}
~~~

### Операции за преобразуване

Операциите за преобразуване или casting операторите нямат тип на връщане, тяхната дефиниция е такава:

``operator <type> () <const>``

Подобни оператори пишем, когато искаме нашият тип да може да се приема и преобразува към друг от типа <type>. Нека напишем преобразуващ оператор на вектора ни към double, които ще представлява магнитута му.

~~~.cpp
// Операторът е член функция!

operator double() const
{
    return sqrt(this->x * this->x + this->y * this->y);
}
~~~

### Операция !

Това е стандартната негация, която използваме в логически изрази. Препоръчва се, щом имплементирате този операотр, да имате съответно и логика за преобразяване на типа към **bool**, което ще рече да имате оператор за преобразяване ``operator bool() const``.

В този пример можете да видите колко елегантна става имплементацията след дефинирането на булев преобразувател...

~~~.cpp
// Операторите са член фунцкии!

operator bool() const 
{
    return this->x != 0 && this->y != 0;
}

bool operator ! () const 
{
    return !(*this);
}
~~~

## Операции ++/--
  
**Префиксен вариант**

Ще разгледаме само оператор ++, тъй като логиката и идеята на -- е абсолютно същата...

Това е **унарна** операция, която променя стойност на типа си и връща обратно новата стойност (по референция)  
Нека напишем префиксен оператор ++, за нашия клас **Vector2D**, който да го мърда с една единица по диагонала на координатната система.

~~~.cpp
Vector2D& operator ++ ()
{
    this->x += 1;
    this->y += 1;
    return *this;
}
~~~

**Постфиксен вариант**

Това
