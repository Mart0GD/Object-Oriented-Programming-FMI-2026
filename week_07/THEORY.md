## Жизнен цикъл на обекти

В предното упражнение се запознахме с това какво представлява повърхностно ООП и каква е идята му. Разгледахме какво са класовете и как се различват от досега познатите ни структурите и видяхме как можем да дефинираме изцяло нови типове чрез тях. Най-важната концепция, с която се запознахме, обаче, беше създаването и зачистването на обекти чрез конструктори и деструктори.

**Как работехме с Конструктори?**

Когато искахме да създадем обект от някакъв тип структура можеше да използваме инициализиращ списък, тъй като всичките елементи по подразбиране в една структура са видими за външния свят и съответно следния синтаксис е много удобен и лесен за използване:

~~~.cpp
struct Circle {
    int radius;
    int center_x;
    int denter_y;
};

Circle tiny_circle = {2,0,0};
~~~

Данните в инициализиращия списък съответстват 1:1 на тези в структурата и по ред и по тип. Това е удобно, обаче си има ограничение, трябва всички членове към които се обръщаме да са видими за нас във външния свят. Тук се намесва проблемът с класовете... 

При класовете този подход с инициализиращия списък няма да ни свърши работа поради две причини: 

- Данните на един клас могат да бъдат поставени в **private** полето му и съответно да са невидими за нас при създаване на обект...
- Може и най-вероятно искаме да се случи някаква специална обработка на тези данни при създаване на обект на класа...

Поради това тук се намесва конструктора. Това беше специална функция без тип на връщане и носеща името на класа, в който се помещава. Конструкторите и функционалностите към тях, които C++ ни предоставя са много и точно те ще бъдат обект на нашето упражнение. 

Ето същия пример, посочен отгоре, но имплементиран чрез клас и няколко overload-a на конструктор, приемащи различен по брой параметри:

~~~.cpp
class Circle{
public:

    // Конструктор по подразбиране, без инициализация на член данни
    Circle() {}

    // Конструктор само с 2 аргумента, извършва се специална логика
    Circle(int radius, int point)
    {
        this->radius = radius;
        this->center_x = this->center_y = point;
    }

    // конструктор, приемащ всички аргументи, които му трябват 
    Circle(int radius, int c_x, int c_y)
    {
        this->radius = radius;
        this->center_x = c_x;
        this->center_y = c_y;
    }

    int diameter() const { return radius * 2; }

// Стойностите ни вече са скрити от външния свят
private:
    int radius;
    int center_x;
    int center_y;
};
~~~

Сега когато се опитаме да създадем обект от тип Circle, компилаторът ще се обърне към дефинираните конструктори и ще се опита да намери подходящ за нашето извикване:

~~~.cpp
int main(void){
    Circle c1();             // това не е конструктор по подразбиране, възприема се като извикване на функция!!
  
    Circle c2;               // това вече ще извика конструкторът по подразбиране ако има такъв...
    Circle c3(1,2,3);        // това ще извика конструкторът, приемащ три параметъра
    Circle c4(1,2,3,4,5,6);  // това ще доведе до грешка, няма дефиниран такъв конструктор 
    return 0;
}
~~~

**Използване на инициализиращ списък**

При извикването на конструктор от какъвто и да е било тип имаме правото да използваме незадължителен инициализиращ списък. Чрез него можем да инициализираме с конкретни стойности член данни на обекта още преди да се е изпълнило тялото на конструктора или да делегираме някой друг конструктор, дали с цел наследяване, дали с цел валидиране на обекта.   
  
Силно се препоръчва този подход на инициализация на данните, тъй като ако искаме да задаем стойности на константни член данни или такива, които са референция към нещо външно, това може да се случи единствено и само в инициализиращия списък...

Ето пример как бихме реализирали някои от горните конструктори с инициализиращ списък:

~~~.cpp
Circle(int radius, int c_x, int c_y)
    : radius(radius)
    , center_x(c_x)
    , center_y(c_y) {  }
~~~

### Видове конструктори

Нека пристъм към разглеждането на различните типове конструктори, които ни предоставя C++

**Конструктор по подразбиране (Default Constructor)**

Това са конструктори, които по дефиниция нямат никакви аргументи, но могат да имат параметри със стойност по подразбиране.

~~~.cpp
class Circle {
public:

    // Конструктор по подразбиране 
    Circle() {
        // логика за инстанциране 
    }

    // Това също е конструктор по подразбиране, но с 3 параметъра със стойности по подразбиране 
    Circle(int agr1 = 0, int arg2 = 1, int arg2 = 2) {
        // логика за инстанциране
    }    
...
};
~~~

Много важна подробност е, че ако един клас няма дефиниран конструктор по подразбиране, то тогава ще му се създаде служебен такъв, който ще използва стойностите по подразбиране, ако има такива, на член данните ни или конструкторите по подразбиране, ако са обекти. 

~~~.cpp
#include <iostream>

class Circle{
public:
    int diameter() const { return radius * 2; }

private:
    int radius  { 1 };
    int center_x{ 0 };
    int center_y{ 0 };
};

int main(void){
    Circle c;
    /*
    *  Създава се обект от тип Circle, за който се извиква служебен inline конструктор по подразбиране.
    *  Стойностите му ще бъдат каквито са зададени в къдравите скоби, а на конзолата ще се принтира 2
    */
    std::cout << c.diameter();
    return 0;
}
~~~

Има случаи, обаче, когато няма да се създаде служебен конструктор по подразбиране.   

Един от тях е ако ние експлицитно окажем, че не искаме да се генерира такъв, това може да стане по два начина или използваме ключовата дума **delete**, или го поставяме в скритата (**private**) секция на нашия клас, като вторият подход се е използвал, преди да се добави запазената дума **delete** и затова първият е за предпочитане.

~~~.cpp
class Circle{
    Circle();           // --> скриване на конструктора от външния свят
public:
...
};

или

class Circle{
public:
    Circle() = delete; // --> експлицитно оказваме, че не искаме такъв конструктор
...
};
~~~

Друг случай, на който може да попаднем е, когато някои от нашите член данни не могат да се конструират по подразбиране. Това се получава ако някое поле е от константен тип или референция и няма зададена стойност в инициализиращия списък на някои от нашите конструктори или ако поле е от тип друг клас и няма конструктор по подразбиране, или деструктор. Тогава нашият служебен конструктор ще бъде дефиниран с `= delete`...

~~~.cpp
// Този клас не може да се инициализира по подразбиране има константно поле id 
class Circle{
public:
...
private:
    const int id;
};

// Това е клас, в който експлицитно е оказано, че не може да се инициализира по подразбиране 
class Point{
public:
    Point() = delete;
private:
    int x, y;
};

// За този клас няма да се генерира конструктор по подразбиране, тъй като Point няма конструктор по подразбиране
class Square{
public:
...
private:
    int area;
    Point center;
};

int main(void){

    Square s; // не е възможно
    Point  p; // и това не е възможно
    Cirlce c; // както и това...

    return 0;
}
~~~
