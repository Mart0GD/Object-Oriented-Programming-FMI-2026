## Жизнен цикъл на обекти

В предното упражнение се запознахме с това какво представлява повърхностно ООП и каква е идята му. Разгледахме какво са класовете и как се различват от досега познатите ни структурите и видяхме как можем да дефинираме изцяло нови типове чрез тях. Най-важната концепция, с която се запознахме, обаче, беше създаването и зачистването на обекти чрез конструктори и деструктори.

**Как работехме с Конструктори?**

Когато искахме да създадем обект от някакъв тип структура можеше да използваме инициализиращ списък, тъй като всичките елементи по подразбиране в една структура са видими за външния свят и съответно следния синтаксис е много удобен и лесен за използване:

~~~.cpp
struct Circle {
    int radius;
    int center_x;
    int denter_y;
};

Circle tiny_circle = {2,0,0};
~~~

Данните в инициализиращия списък съответстват 1:1 на тези в структурата и по ред и по тип. Това е удобно, обаче си има ограничение, трябва всички членове към които се обръщаме да са видими за нас във външния свят. Тук се намесва проблемът с класовете... 

При класовете този подход с инициализиращия списък няма да ни свърши работа поради две причини: 

- Данните на един клас могат да бъдат поставени в **private** полето му и съответно да са невидими за нас при създаване на обект...
- Може и най-вероятно искаме да се случи някаква специална обработка на тези данни при създаване на обект на класа...

Поради това тук се намесва конструктора. Това беше специална функция без тип на връщане и носеща името на класа, в който се помещава. Конструкторите и функционалностите към тях, които C++ ни предоставя са много и точно те ще бъдат обект на нашето упражнение. 

Ето същия пример, посочен отгоре, но имплементиран чрез клас и няколко overload-a на конструктор, приемащи различен по брой параметри:

~~~.cpp
class Circle{
public:

    // Конструктор по подразбиране, без инициализация на член данни
    Circle() {}

    // Конструктор само с 2 аргумента, извършва се специална логика
    Circle(int radius, int point)
    {
        this->radius = radius;
        this->center_x = this->center_y = point;
    }

    // конструктор, приемащ всички аргументи, които му трябват 
    Circle(int radius, int c_x, int c_y)
    {
        this->radius = radius;
        this->center_x = c_x;
        this->center_y = c_y;
    }

    int diameter() const { return radius * 2; }

// Стойностите ни вече са скрити от външния свят
private:
    int radius;
    int center_x;
    int center_y;
};
~~~

Сега когато се опитаме да създадем обект от тип Circle, компилаторът ще се обърне към дефинираните конструктори и ще се опита да намери подходящ за нашето извикване:

~~~.cpp
int main(void){
    Circle c1();             // това не е конструктор по подразбиране, възприема се като извикване на функция!!
  
    Circle c2;               // това вече ще извика конструкторът по подразбиране ако има такъв...
    Circle c3(1,2,3);        // това ще извика конструкторът, приемащ три параметъра
    Circle c4(1,2,3,4,5,6);  // това ще доведе до грешка, няма дефиниран такъв конструктор 
    return 0;
}
~~~

**Използване на инициализиращ списък**

При извикването на конструктор от какъвто и да е било тип имаме правото да използваме незадължителен инициализиращ списък. Чрез него можем да инициализираме с конкретни стойности член данни на обекта още преди да се е изпълнило тялото на конструктора или да делегираме някой друг конструктор, дали с цел наследяване, дали с цел валидиране на обекта.   
  
Силно се препоръчва този подход на инициализация на данните, тъй като ако искаме да задаем стойности на константни член данни или такива, които са референция към нещо външно, това може да се случи единствено и само в инициализиращия списък...

Ето пример как бихме реализирали някои от горните конструктори с инициализиращ списък:

~~~.cpp
Circle(int radius, int c_x, int c_y)
    : radius(radius)
    , center_x(c_x)
    , center_y(c_y) {  }
~~~

### Видове конструктори

Нека пристъм към разглеждането на различните типове конструктори, които ни предоставя C++

**Конструктор по подразбиране (Default Constructor)**

Това са конструктори, които по дефиниция нямат никакви аргументи, но могат да имат параметри със стойност по подразбиране.

~~~.cpp
class Circle {
public:

    // Конструктор по подразбиране 
    Circle() {
        // логика за инстанциране 
    }

    // Това също е конструктор по подразбиране, но с 3 параметъра със стойности по подразбиране 
    Circle(int agr1 = 0, int arg2 = 1, int arg2 = 2) {
        // логика за инстанциране
    }    
...
};
~~~

Много важна подробност е, че ако един клас няма дефиниран конструктор по подразбиране, то тогава ще му се създаде служебен такъв, който ще използва стойностите по подразбиране, ако има такива, на член данните ни или конструкторите по подразбиране, ако са обекти. 

~~~.cpp
#include <iostream>

class Circle{
public:
    int diameter() const { return radius * 2; }

private:
    int radius  { 1 };
    int center_x{ 0 };
    int center_y{ 0 };
};

int main(void){
    Circle c;
    /*
    *  Създава се обект от тип Circle, за който се извиква служебен inline конструктор по подразбиране.
    *  Стойностите му ще бъдат каквито са зададени в къдравите скоби, а на конзолата ще се принтира 2
    */
    std::cout << c.diameter();
    return 0;
}
~~~

Има случаи, обаче, когато няма да се създаде служебен конструктор по подразбиране.   

Един от тях е ако ние експлицитно окажем, че не искаме да се генерира такъв, това може да стане по два начина или използваме ключовата дума **delete**, или го поставяме в скритата (**private**) секция на нашия клас, като вторият подход се е използвал, преди да се добави запазената дума **delete** и затова първият е за предпочитане.

~~~.cpp
class Circle{
    Circle();           // --> скриване на конструктора от външния свят
public:
...
};

или

class Circle{
public:
    Circle() = delete; // --> експлицитно оказваме, че не искаме такъв конструктор
...
};
~~~

Друг случай, на който може да попаднем е, когато някои от нашите член данни не могат да се конструират по подразбиране. Това се получава ако някое поле е от константен тип или референция и няма зададена стойност в инициализиращия списък на някои от нашите конструктори или ако поле е от тип друг клас и няма конструктор по подразбиране, или деструктор. Тогава нашият служебен конструктор ще бъде дефиниран с `= delete`...

~~~.cpp
// Този клас не може да се инициализира по подразбиране има константно поле id 
class Circle{
public:
...
private:
    const int id;
};

// Това е клас, в който експлицитно е оказано, че не може да се инициализира по подразбиране 
class Point{
public:
    Point() = delete;
private:
    int x, y;
};

// За този клас няма да се генерира конструктор по подразбиране, тъй като Point няма конструктор по подразбиране
class Square{
public:
...
private:
    int area;
    Point center;
};

int main(void){

    Square s; // не е възможно
    Point  p; // и това не е възможно
    Cirlce c; // както и това...

    return 0;
}
~~~

Посленият случай, в който може на наш клас да не се генерира служебен конструктор по подразбиране е, когато имаме дефиниран както и да е било конструктор, който не по подразбиране.

Ето пример:

~~~.cpp
class Circle{
public:

    /*
    *  Имаме дефиниран конструктор с 3 аргумента
    *  => няма да получим конструктор по подразбиране...
    */
    Circle(int radius, int c_x, int c_y)
        : radius(radius)
        , c_x(c_x)
        , c_y(c_y) {}

private:
    int radius;
    int c_x, c_y;
};

int main(void) {
    Circle c; // няма да проработи
    return 0;
};
~~~

**Копиращ Конструктор (Copy Constructor)**

Копиращите конструктори са неразделна част от жизненият цикъл на един обект, но най-вероятно досега не сте се сблъсквали с тази концепция ако сте писали на други езици, тъй като по-голямата част от логиката остава скрита от пишещия, но в C++ имаме честта сами да ръководим как нашите данни биват копирани между обекти, за да си подсигурим правилен инвариант и собстевност на данните между обекти от един и същи тип. 

Копиращия конструктор на един обект бива извикан при инициализация на обект чрез друг или при експлицитно създаване на обект с кръгли скобки и подаден аргумент друг обект. Ето няколко пример:

~~~.cpp
class Circle{
public:
    Circle()  : radius(1), c_x(0), c_y(0) {};
    Circle(int radius, int c_x, int c_y) : radius(radius), c_x(c_x), c_y(c_y) {};

private:
    int radius;
    int c_x, c_y;  
};

int main(void) {
    Circle c1;
    Circle c2(10,1,1);

    // Това представлява извикване на копиращия конструктор на Circle за инстанциране на обекта c3, чрез данните на обекта c1
    Circle c3 = c1;

    // Това е абсолютно същото извикване като горното, но тук извикваме конструктора експлицитно с обекта c2
    Circle c4(c2);

    return 0;
};
~~~

И в двете извиквания, фигуриращи в примера се извиква служебния копиращ конструктор зададен на класа Circle, който в случая прави така наречено плитко копиране (**shallow copy**) на данните на обекта, който се копира. Така че логиката ни е стабила, поради факта че нашите обекти от тип Circle се състоят само от скаларни по тип данни. 

Обаче в момента, в който добавим каквито и да е динамични член данни към класа Circle, тази логика няма да бъде коректна вече, тъй като плиткото копиране ще ни доведе до споделена собственост на динамичните данни между двата обекта - копирания и копиращия. Това може да доведе до много сериозни проблеми ако не е умишлено, като например двойно изтриване, хвърчащи референции и указатели и в най-лошия случай **use after free** грешки. 

Ето конкретен пример с това, кога служебният копиращ конструктор няма да ни свърши работа:

~~~.cpp
#include <string.h>
#include <new>

class Book {
public:
    Book() : text(nullptr) {};
    Book(const char* text)
        : text(nullptr)
    {
        if(!text) ; // some error...

        this->text = new(std::nothrow) char[strlen(text) + 1];
        if(!this->text) ; // some error

        strcpy(this->text, text);
    };    

    ~Book() 
    { 
        delete[] text; 
        text = nullptr;
    }

private:
    char* text; 
};

int main(void){

    Book bible("And there was light!");

    {
        // Извикване експлицитно на служебен копиращ конструктор 
        Book necronomicon(bible); 
    } // --> край на блока, извиква се деструктор на necronomicon

    // Какво става тук??
    // necronomicon изтри данните на bible...
    // Това изобщо не е желан ефект

    return 0;
};
~~~

При такива ситуации имаме нужда от това да си дефинираме наш собстевн копиращ конструктор. Но преди да ви покажа как става това нека рагледаме какъв вид точно има копиращия конструктор и как можем да го дефинираме...

Имаме следния синтаксис, когато дефинираме копиращи конструктори:

~~~.cpp
Circle(Circle& other); // Това е позволено, но не го използвайте, защото позволява промяна на обекта, от който копираме, а това за нашите цели няма да се очаква!!!
Circle(const Circle& other); // Това е конструктора, който главно ще използваме 

// От тези двата ще се абстархираме, за сега просто информативно да знаете, че съществуват 
Circle(volatile Circle& other);
Circle(volatile const Circle& other);

// Позволяват се допълнителни параметри, но само със стойности по подразбиране, няма да го срещнете често...
Circle(Circle& other, int i = 42, string label = "Box");
~~~

Това представлява прототипа на всеки копиращ конструктор, като може да имаме няколко копиращи конструктора, стига да са с различна дефиниця от показаните. Всеки клас има служебен копиращ конструктор от втория тип, който прави плитко копие. 

Нека сега разпишем примера отново, но този път правилно...

~~~.cpp
#include <string.h>
#include <new>

class Book {
public:
    Book() : text(nullptr) {};
    Book(const char* text)
        : text(nullptr)
    {
        if(!text) ; // some error...

        this->text = new(std::nothrow) char[strlen(text) + 1];
        if(!this->text) ; // some error

        strcpy(this->text, text);
    };    

    // Дефинираме копиращ конструктор
    Book(const Book& other)
    {
        text = new(std::nothrow) char[strlen(other.text) + 1];
        if(!text) ; // some error

        strcpy(text, text);
    }

    // Забелязахте ли колко много логиката си прилича с конструктора? Нека преизползваме малко логика (;
    // Book(const Book& other)
    //     : Book(other.text) {};

    ~Book() 
    { 
        delete[] text; 
        text = nullptr;
    }

private:
    char* text; 
};

int main(void){

    Book bible("And there was light!");

    {
        // Извикване експлицитно на служебен копиращ конструктор 
        Book necronomicon(bible); 
    } // --> край на блока, извиква се деструктор на necronomicon

    // Какво става тук??
    // necronomicon изтри своето собстевно копие на данните на bible
    // Това вече е желан ефект => Успех!!

    return 0;
};
~~~

Това е много удобна функционалност, която ни позволява да дефинираме каквито си искаме типове и да контролираме техните взаимоотношения по времето на нашата програма.
 
Има няколко важни подробности, които трябва да знаете. Както при конструкторите по подразбиране така и тук може да не получим служебен копиращ конструктор, като причините са доста сходни:

- Експлицитно сме го изтрили с `= delete`...
- Имаме член данни, които не могат да се копират (нямат дефиниран какъвто и да е копиращ конструктор...)

Ето някои примери:

~~~.cpp
#include <iostream>

class Point {
public:
    Point(int x, int y) : x(x), y(y) {};
    Point(const Point& other) = delete;   // експлицитно изтрит копиращ конструктор
private:
    int x, y;
};

class Circle{
public:
    Circle(std::istream& is = std::cin) : is(is), radius(10), cords({0,0}) {};
private:
    std::istream& is; // имаме референция към файл, това е позволено, тъй като референцията просто ще се прехвърли

    int radius;
    Point cords;      // това обаче ще изтрие служебния копиращ конструктор на Circle...
};

int main(void){
    Circle c1;
    Point p1(1,1);

    Circle c2 = c1;
    Point p2 = p1;

    return 0;
};
~~~

**Move Constructor**

Това е специален тип конструкор, който се добавя към езика във версия C++ 14 и по принцип не се очаква от вас да го използвате на каквито и да е изпити ако се интересувата от тези неща разбира се. Иначе като оставим настрана академичните страдания, почти на 100% задължително е да знаете както е това, защото на практика се използва постоянно. Move семантиката на C++ ни позволява да преместваме собстеност на данни между обекти от един и същи тип без да се копира. Най-интуитивно можете да си представите, че единият обект краде данните на другия умиращ от старост обект, защото вече не са му нужни. Този конструктор се използва върху RValue reference стойности или на български, временни обекти. Това са обекти, които не са закачени към променлива и им предстои извикване на деструктор. 

Ето това е пример за RValue reference:

~~~.cpp
struct point {
    int x, y;
};

void foo(const point&& rvalue_ref);

int main(void)
{

}
~~~
