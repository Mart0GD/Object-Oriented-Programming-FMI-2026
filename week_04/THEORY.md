## Леко обобщение до момента...

В предишното упражнение се занимавахме с файлове с последователен достъп. Това означаваше, че данните му се четат последователно от началото на файла 
до неговият край. Това реализирахме като към някакъв текстови файл асоциирахме поток от **fstream** в стандартен (**текстови**) режим.     
    
Можехме ли да **добавим** диркетно информация на дадена позиция - **уви не...**  
  
1) Можем да презапишем информацията от файла в нов, да извършим корекцията и да допишем останалото, като изтрием 
първоначалния файл. Скъпа операция, но работеща.  
2) Мърдаме всички записи с едно назад, както правехме при изтриването в масиви по УП, но постоянното преместване на указателя в текстови 
файл е скъпо, поради нуждата да заредим файла наново и да отместим указателя спрямо началото при всяко стъпка.

Можем ли да **редактираме** данни директно на дадена позиция - **отново не...**    

1) Ако данните за редакция са с еднакъв размер, презаписваме върху старото съдържание.
2) Ако данните ни не са толкова перфетни, можем единствено да презапишем наново файла, като извършим редакцията на желаното от нас място, тъй като при подход 1 ще 
трябва да пишем върху данни, които не искаме да редактираме 

Можем ли да **изтрием** данни от файл с последователен достъп - **както вече се досещате, не...**  
  
1) Изчитаме файла докато прескочим желаната позиция за изтриване и да мърдаме последователно съдържанито с едно назад.  
2) Създаваме нов файл, презаписваме съдържанието до желаната за изтриване позиция, пропускаме съдържанието, което не искаме и презаписваме останалото.  

Както виждате файловете с последователен достъп, не ни устройват особено много, когато искаме да извършваме операции за редактиране, изтриване или добавяне. Затова 
нека въведем една нова концепция.

## Файлове с директен достъп

Както знаем файловете с директен достъп ни позволяват да прескачаме на прочизволни места, без нуждата да се изчитат данните последователно. Тук нямаме задължителен 
формат за всеки запис да бъде с големина 1 байт. Записите ни са **произволни по размер** и **неформатирани**. Затова имаме нужда от някакви външни знания за формата.
C++ не ни гарантира такива под никаква форма, затова ние сами и ще си ги определяме. В пределите на курса ще се сблъскаме с различни подходи за формат, както например:

  - Файлът да е разделен на блокове с еднакъв размер.
  - Преди всеки запис да има метаданни, които оказват размера му.
  - Всеки запис във файла да е с еднакъв размер.
  - и още много...

## Работа с файлове с директен достъп в C++  
   
### Отваряне 
  
Както при текстовите файлове и тук ще искаме да асоцираме даден файл към поток, за да получим буфер, от който да можем да четем и записваме. За работа с файлове
с директен достъп отново ще изполваме **fstream**, като едиснтвената разлика с текстовите файлове е, че тук раборим в двоичен (**binary**) режим. Всеки файл в двоичен 
режим е файл с директен достъп.

~~~.cpp
// Отваряне на файл с директен достъп
#include <fstream>

int main(void){

  // Отваряме поток за запис или четене в двоичен режим
  std::ofstream os("File.bin", std::ios::binary);
  if(!os.is_open()) return -1;
  
  return 0;
}
~~~

### Четене и писане 

При двоичните файлове четенето и писането с операторите `>>`, `<<` и функцията `getline()` **не е ефективно**, тъй като:

- не искаме **форматирано четене/писане**
- данните **най-често не са записани последователно като текст**

Затова използваме функциите:

- `istream& read(char* s, streamsize n);` – за четене на сурови байтове
- `ostream& write(const char* s, streamsize n);` – за запис на сурови байтове

Функциите са дефинирани в:

- `<istream>` – за `read()`
- `<ostream>` – за `write()`

Когато работим с `fstream`, имаме достъп **и до двете функции**, защото той комбинира входен и изходен поток.

~~~.cpp
// Пример за четене от двоичен файл

#include <iostream>
#include <fstream>

int main () {
  streampos size;
  char * memblock;

  std::ifstream file ("example.bin", ios::in|ios::binary|ios::ate);
  if (file.is_open())
  {
    size = file.tellg();
    memblock = new(std::nothrow) char [size];
    if(!memblock) return -1;

    file.seekg (0, ios::beg);
    file.read (memblock, size);
    if(!file) { std::cout << "error!; return -1; }
    file.close();

    cout << "the entire file content is in memory";
    delete[] memblock;
  }
  else cout << "Unable to open file";

  return 0;
}
~~~

~~~.cpp
// Пример за запис в двоичен файл

#include <iostream>
#include <fstream>

int main() {

    std::fstream s("File.txt", std::ios::binary | std::ios::in | std::ios::out | std::ios::trunc);
    if(!s.is_open()) return -1;

    char text[] = "Length is 13";
    int length = sizeof(text);

    s.write(reinterpret_cast<const char*>(&length), sizeof(int));
    s.write(text, length);

    if(!s) {
        std::cout << "error";
        return -1;
    }

    // какво има във файла
    // |4 байта число|13 байта за низа|

    int size;
    s.seekg(0, std::ios::beg);

    s.read(reinterpret_cast<char*>(&size), sizeof(size));
    if(!s) return -1;

    char* str = new(std::nothrow) char[size];
    if(!str) return -1;

    s.read(str, size);
    if(!s) return -1;

    delete[] str;
    s.close();

    return 0;
}
~~~
